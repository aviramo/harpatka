# יוצאים להרפתקה — Landing Page

דף נחיתה לאירוע הכרויות חווייתי ("יוצאים להרפתקה"). אתר **סטטי**: `index.html` + `images/`. Tailwind via CDN, Vanilla JS, RTL.

> ⚠️ הערכים הרגישים (endpoint, מזהה גיליון, טלפונים, מזהי טראקינג) **לא** נשמרים כאן. הם חיים ב-`CONFIG` שבתוך `index.html`, ובסיכום מקומי `.claude/secrets.md` (לא נדחף ל-repo).

## פרסום (Deploy)
- **GitHub Pages**, repo `aviramo/harpatka`, ענף `main`.
- דומיין מותאם (apex): **https://harpatka.co.il/** (מוגדר דרך קובץ `CNAME` בשורש ה-repo + רשומות DNS). `https://aviramo.github.io/harpatka/` ממשיך לעבוד ומפנה לדומיין.
- לפרסם שינוי: `git add -A && git commit && git push origin main` (gh CLI לא מותקן; דוחפים ישירות עם git, אישורי GitHub שמורים ב-Credential Manager).
- `.claude/` ב-`.gitignore` (לא נדחף).

## טופס ההרשמה → Google Sheets
- שליחה ב-`fetch` POST (`mode:'no-cors'`, `Content-Type: text/plain`) ל-**Google Apps Script Web App** (ה-URL ב-`CONFIG.sheetsEndpoint`).
- הסקריפט (doPost) כותב שורה לטאב התשובות. **עמודות:** A=חותמת זמן (שעון ישראל, dd/MM/yyyy HH:mm:ss) · B=שם מלא · C=מגדר · D=טלפון (טקסט עם גרש; מפתח כפילות) · E=גיל · F=סטטוס (ריק; מנוהל ידנית).
- **מניעת כפילות:** ב-Apps Script לפי עמודה D. אם קיים — לא נכתב, אבל המשתמש עדיין רואה מסך הצלחה.

## ולידציית טלפון
- ניקוי רווחים/מקפים/סוגריים, ואז Regex:
  `^(\+972[1-9]\d{8}|0(5\d|7\d)\d{7}|0[2-489]\d{7}|\+[1-9]\d{9,14})$`
- **נורמליזציה לאחסון:** `+972`→`0` ; מקומי `0...` נשאר ; קידומת בינ"ל אחרת נשמרת עם `+`.

## אינטגרציות (ערכים ב-`CONFIG`)
- **Meta (Facebook) Pixel:** `PageView` בטעינה, `track('Lead')` בשליחה מוצלחת.
- כפתור "לעדכונים" / "הצטרפות לקבוצה" → קישור קבוצת וואטסאפ.
- כפתור "בקשת קישור לתשלום מענבר" → קישור `wa.me` עם הודעה מוכנה.

## כללי טופס / UX
- שם מלא: חייב רווח (שם פרטי + משפחה). מגדר: שני כפתורים אישה/גבר (♀/♂). גיל: 18–70.
- כפתור "שליחה" **תמיד פעיל ובולט** (אין `disabled` לפי ולידציה); הולידציה נאכפת בזמן ה-submit — לחיצה כשלא תקין מסמנת את כל השדות כ"נגעו", מציגה שגיאות ולא שולחת. הכפתור מתעמעם רק לרגע בזמן השליחה עצמה ("שולח..."). שדה לא תקין נצבע באזהרה אחרי נגיעה.
- מילוי-אוטומטי של הדפדפן: כלל CSS על `input.field:-webkit-autofill` מנטרל את הרקע הלבן (box-shadow פנימי אטום `#0a1e29` + `-webkit-text-fill-color`) כדי לשמר את העיצוב הכהה.
- מסך תודה: כותרת "הפרטים נרשמו" עם 🎉 מעליה. ההצלחה נזכרת ב-`sessionStorage` (`regDone`) — לא ב-URL — כדי שרענון אצל מי שנרשם יציג שוב את מסך התודה, אבל **קישור משותף תמיד מציג את הטופס**. בשליחה מנקים פרמטר `status` ישן מה-URL אם קיים. כפתור **X** (`#backToForm`) בפינת מסך התודה סוגר וחוזר לטופס (מסיר את `regDone`).
- סטריפ המילים תחת "מה בתכנית" נע ברצף (ltr-wrap, ‎-50%→0).
- **פיסוק — נקודות בסוף משפט:** נקודה מוצדקת **רק** אם המשפט נגמר ויש אחריו מילה/משפט נוסף **באותה שורה** (שורה מוצגת, אחרי ש-HTML מכווץ מעברי־שורה במקור לרווח). נקודה שסוגרת שורה מוצגת — לפני `<br>`, או לפני סגירת פסקה/פריט `</p>`/`</li>`/`</span>` **גם אם הסגירה נמצאת בשורת המקור הבאה** — **לא** אמורה להופיע. שים לב: לא מספיק לחפש `.<`; יש לתפוס גם נקודה שאחריה מעבר־שורה במקור ואז תג סוגר. נקודות באמצע שורה שאחריהן ממשיך טקסט (כמו "לא. אפשר…" / "מושלם. זה ערב אנושי") נשמרות.
- סקשן `#why` ("מהי הרפתקה?") — טקסט ממורכז בלבד (בלי וידאו/תמונה). *(הוסר סרטון ה-motion שהיה כאן וכל ה-JS שניהל אותו: `#motionVid`, `#motionToggle`, `#motionMuted`, `#motionFrame`, `#motionHome`, `#motionMobileSlot`, וה-IntersectionObserver על `#afterMotionImg`. הקובץ `images/motion.mp4` כבר לא בשימוש.)*
